#### 通信协议
1. `概述`：
   通信协议：发送信息/接收信息的规则、规范。就像寄快递，要制定邮寄规则和规范。
   没有协议，两台机器根本看不懂对方发的 01 二进制数据，就像两个人一个说中文、一个说日语，没有翻译完全无法沟通。
   TCP/IP 四层模型，每一层都有自己独立的协议，只管自己层级的规则，互不干扰
2. `形象类比`：两个人要正常通话，必须遵守一堆默认规则（这就是 “人与人之间的通信协议”）：
   1. 先拨号，对方接通才能说话（建立连接规则）
   2. 轮流说话，不能同时大喊（收发时序规则）
   3. 听得见听不懂要重复一遍（出错重传规则）
   4. 说完说 “挂了” 再挂断（断开连接规则）
3. `一套通信协议包含哪些约定`
   计算机的通信协议，就是给机器定好的同类规则，只不过是用二进制、固定格式规定死
   1. 数据长什么样：数据开头标记、数据长度、字段顺序（比如 Dubbo 魔数、HTTP 的 GET/POST 格式）
   2. 怎么建立连接：要不要先握手（TCP 三次握手）
   3. 怎么发送、接收：一次发多大包、顺序怎么保证
   4. 出错怎么办：丢包重发、损坏丢弃
   5. 怎么结束通信：断开连接流程


#### 计算机网络分层(TCP/IP 四层模型)
每层分工明确，共同实现计算机网络数据传输。类似寄快递，寄件人、收揽人、交通运输分工明确。
1. `应用层`
   1. `应用层通信协议`：（管 “内容格式”）
      HTTP、Dubbo协议、Nacos、DNS、Triple (gRPC)
      约定：消息里业务内容怎么写
      例：HTTP 规定第一行写 GET /xxx，Dubbo 规定开头是 0xdabb 魔数
2. `传输层`
   1. `传输层通信协议`：（管 “分发、可靠传输”）
      TCP(可靠)、UDP(高速无保障)
      约定：端口标识、连接、丢包处理、分包重组
3. `网络层(网际层)`
   1. `网络层通信协议`：（管 “寻址指路”）
      IP协议
      约定：IP 地址格式、跨机器转发路线
4. `网络接口层(链路+物理)`
   1. `网络接口层通信协议`：（管 “硬件传信号”）
      以太网协议 MAC、电 / 光传输规范
      约定：MAC 地址、网线二进制电信号传输规则

上层协议依赖下层协议：
1. 比如 HTTP（应用协议）自己不能传输数据(http没有通信能力)，必须交给 TCP（传输协议）去发送；TCP 又要靠 IP 寻址。
2. http就是应用之间的通信协议 在应用层 应用之间的通信靠http http依靠下一层如tcp
3. 形象类比：
   把 HTTP 比作书信格式规范
   TCP 相当于快递公司服务规则
   IP 是全国地址系统，负责跨城市找到目标机器
   浏览器写一封符合 HTTP 格式的信 -> 交给 TCP 快递公司打包运输 -> IP 规划路线送到对方服务器
协议是双方统一标准：
客户端和服务端必须用一模一样的协议，一个用 Dubbo、一个用 HTTP，完全无法通信。


#### TCP协议
1. 传输层协议
2. 负责两台机器建立一条数据通道，传输二进制数据。
3. 连接分两种：
   1. 短连接
      流程：建立连接 -> 发一次请求响应 -> 立刻断开连接
      每次交互都要重新握手、挥手，频繁调用会大量消耗网络、CPU。
      HTTP/1.1 默认短连接模式就是这种
   2. 长连接
      一条 TCP 连接建立后，不马上断开，可以反复发送多次请求、接收多次响应，复用这条通道。
      Dubbo 默认 dubbo 协议就是纯 TCP 长连接:
      1. 客户端和服务端只握手 1 次，打通一条 TCP 通道
      2. 后续上千次接口调用都复用这条连接，不用重复建连断连
      3. 长时间无数据才自动断开，有请求继续复用
         优点：减少握手开销，内网高频 RPC 调用速度更快、性能更高。
4. TCP三次握手四次挥手


#### 常见网络概念
1. TCP协议特性：
   面向连接、可靠传输、有序、字节流、三次握手 / 四次挥手、滑动窗口、粘包拆包、拥塞控制
2. Socket：TCP 通信端点（IP + 端口），Dubbo 服务提供者绑定 Socket 监听，消费者创建 Socket 连接
3. 序列化：应用层对象->二进制字节流（网络只能传字节，不能直接传 Java 对象）
4. IO 模型：BIO/NIO/AIO，Dubbo 底层 Netty（NIO 多路复用）
5. 半包 / 粘包：TCP 是无边界字节流，必须自定义报文长度头解决，Dubbo 协议自带定长包头
6. 长连接/短连接
   长连接：一次 TCP 连接复用多次请求（Dubbo 默认）
   短连接：每次请求新建 TCP（HTTP1.0）
7. 端口、寻址、负载均衡：配合 Nacos 注册中心获取服务 IP 端口，网络层 IP 路由分发请求


#### 计算机网络 IO
1. BIO：一连接一线程，高并发下线程爆炸，操作系统内核切换开销大
2. NIO（Netty）：多路复用 Selector，单线程轮询所有 Socket 连接，少量线程管理上万 TCP 长连接，Dubbo 底层默认
3. AIO：异步非阻塞，极少使用

#### HTTP&HTTPS
1. HTTP：超文本传输协议，明文传输，端口 80
   裸奔明文，速度快，无安全，仅适合纯静态公开资讯
2. HTTPS：HTTP + SSL/TLS 加密层，密文传输，端口 443
   防数据被篡改
   中间人无法劫持
   套上 SSL/TLS 加密外壳，加密防窃取、证书防冒充，所有业务网站、支付、登录必须使用
